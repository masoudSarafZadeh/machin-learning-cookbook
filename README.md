# Student Performance Predictive Pipeline & Advanced Ensemble Optimization 🚀📊
**ALIN Datase:** https://ieee-dataport.org/documents/alin-open-dataset-math-adaptive-learning
An end-to-end, production-grade machine learning pipeline designed to predict student performance (`averagedCorrectness`) from continuous behavioral metrics. This repository showcases the transition from thorough exploratory analysis and statistical data cleaning to complex unsupervised feature synthesis, dimensionality reduction, and hyperparameter optimization.

---

## Core Engineering Highlights

* **Rigorous Statistical Outlier Mitigation:** Employs an automated Interquartile Range (IQR) validation filter ($1.5 \times \text{IQR}$) across the entire continuous sample spectrum, dynamically handling skewed distribution tails and replacing anomalous variances using rolling column means.
* **Leakage-Free Feature Engineering (Strict Separation):** To eliminate lookahead bias and validation data contamination, feature scalers (**StandardScaler**), dimensionality constraints (**PCA**), and spatial cluster profiles (**K-Means**) are computed sequentially and exclusively fitted *only* on training splits before transforming evaluation arrays.
* **PCA Dimensionality Reduction:** Maximizes computational efficiency by deploying Principal Component Analysis (PCA) calibrated to dynamically preserve $95\%$ of the absolute cumulative dataset variance, trimming redundant features while retaining essential signals.
* **Unsupervised Feature Injection:** Leverages K-Means clustering natively within the modeling stream. The optimal structural coordinates generated by spatial cluster centers are dynamically appended back to the regression feature matrix as synthetic high-value descriptors.
* **Extensive Multi-Model Benchmarking:** Evaluates a broad suite of 10 structural estimators concurrently, comparing classical regularized expressions (Linear Regression, Ridge, Lasso, ElasticNet) and distance/boundary networks (KNN, SVR) against robust ensemble heavyweights (Random Forest, XGBoost, LightGBM).
* **Advanced MLOps Hyperparameter Tuning:** Maximizes prediction margins by running a distributed **`RandomizedSearchCV`** loop across 8 separate operational dimensions of LightGBM, utilizing maximum local thread parallelization (`n_jobs=-1`).

2. **Configure Dependencies:**
Isolate this pipeline inside a clean local virtual environment:
```Bash

python -m venv venv
source venv/bin/activate  # On Windows terminal use: venv\Scripts\activate

pip install -r requirements.txt
```

Ensure your primary data file `student_data_processed.csv` is anchored directly within the root project folder alongside `main.py`.

## Pipeline Execution
Run the complete optimized workflow directly from your shell console:
```Bash
python main.py
```
## Execution Lifecycles

* **Ingestion & In-Memory Preprocessing:** Loads feature targets and implements mathematical boundary shifts to prune extreme outliers.
* **Visual Assessment:** Generates structural correlation arrays and saves verification images directly to the disk asset system.
* **Data Splitting & Transformation:** Divides continuous segments, calculates scalar variance metrics, executes PCA reduction, appends unsupervised spatial cluster targets, and updates data shapes.
* **Phase 1 Benchmark:** Runs evaluations across all 10 algorithms simultaneously, instantly compiling and outputting an ordered leaderboard to the terminal window.
* **Phase 2 Tuning:** Launches a distributed random grid sweep to optimize hyperparameter settings for the highest-performing ensemble regressor.
